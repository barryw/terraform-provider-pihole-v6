# pihole Provider

The PiHole provider allows you to manage [Pi-hole](https://pi-hole.net/) v6 configuration as infrastructure-as-code. It communicates with the Pi-hole v6 REST API to manage DNS records, CNAME records, groups, adlists, domain lists, client definitions, and configuration settings.

This provider targets **Pi-hole v6 exclusively** and requires an app-password for authentication.

## Prerequisites

Before using this provider, you must configure two settings on each Pi-hole instance:

### 1. Create an App-Password

In the Pi-hole web UI, go to **Settings > API > App password** and generate a new app-password.

### 2. Enable app_sudo

By default, app-password sessions are read-only. You must enable app_sudo for write operations:

**Via CLI:**
```bash
sudo pihole-FTL --config webserver.api.app_sudo true
```

**Via the web UI:** Settings > API > "Allow app-password authenticated sessions to extend sudo"

~> **Important:** Without app_sudo enabled, all write operations fail with 403 Forbidden. This is a separate setting from allow_destructive.

## Example Usage

```terraform
terraform {
  required_providers {
    pihole = {
      source = "barryw/pihole-v6"
    }
  }
}

# Configure the PiHole provider
provider "pihole" {
  url      = "http://192.168.1.1:8080"
  password = var.pihole_password
}

# Multiple instances using aliases
provider "pihole" {
  alias    = "secondary"
  url      = "http://192.168.1.2:8080"
  password = var.pihole_password
}

variable "pihole_password" {
  type      = string
  sensitive = true
}
```
